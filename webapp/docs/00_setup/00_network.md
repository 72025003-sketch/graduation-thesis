# 船内ネットワークアーキテクチャ

本演習環境は、実際の船舶ネットワークにおけるセキュアな設計（ゾーン分割）を模倣して構築されています。
演習を進める前に、自分がどのネットワーク（セグメント）にいて、どこへ向かって攻撃を仕掛けるのかを理解してください。

---

## 1. ネットワークの全体像 (Network Topology)

本環境は、セキュリティレベルの異なる4つの内部ネットワークと、模擬インターネットから構成されています。
各ゾーンはルータ/ファイアウォール（FW1, FW2）によって隔てられています。

```text
[Internet] (192.168.1.0/24)
   │
[ FW1 ] (エッジルータ)
   │
   ├─ [Crew NW] (192.168.5.0/24) ──★ あなた(C2-Server)はここ！
   │     └─ c2-server (192.168.5.30)
   │     └─ crew-pc   (192.168.5.50)
   │
   └─ [ DMZ ] (10.254.0.0/24)
         │
      [ FW2 ] (内部ルータ)
         │
         ├─ [Business NW] (192.168.10.0/24)
         │     └─ log-server (192.168.10.20)
         │     └─ webapp     (192.168.10.50)
         │
         └─ [Navi NW] (192.168.20.0/24) ──★ 最終ターゲット！
               └─ gps-ais    (192.168.20.11)
               └─ opencpn    (192.168.20.20)
```

---

## 2. 各ネットワークゾーンの役割

### Crew NW (乗組員ネットワーク / 192.168.5.0/24)

* **セキュリティレベル:** 低
* 乗組員が個人的なスマートフォンやPCを接続するネットワークです。インターネットへの接続口があり、私用デバイスが接続されるためマルウェア感染のリスクが最も高い領域です。
* **攻撃の起点（スタート地点）** となります。

### Business NW (業務ネットワーク / 192.168.10.0/24)

* **セキュリティレベル:** 中
* 船舶の運航管理や各種ログ収集のためのネットワークです。
* 今回の演習では、ここに配置されている **Log Server (192.168.10.20)** が最初の中間ターゲット（踏み台）となります。

### Navi NW (航海機器ネットワーク / 192.168.20.0/24)

* **セキュリティレベル:** 高
* 船舶の安全な航行に直結する非常に重要なネットワーク（OTネットワーク）です。本来は他から完全に隔離されているべきですが、業務効率化などの理由で Business NW と繋がってしまっている構成です。
* **最終目標:** ここに存在する航海機器（ECDISやGPS）に対してスプーフィング攻撃を仕掛け、船の航路を操ることが目的です。

---

## 3. 演習の目標（ラテラルムーブメント）

本演習の最終的な目標は、**「低セキュリティ領域（Crew NW）から高セキュリティ領域（Navi NW）へ侵入し、航海機器をハッキングすること」** です。

1. **偵察:** C2 Serverからネットワークをスキャンし、到達可能な脆弱な端末を探します。
2. **踏み台の確保:** 脆弱なパスワード等を利用し、Business NW の `log-server` へ侵入します。
3. **OT領域への攻撃:** 乗っ取った `log-server` を踏み台にして、Navi NW の機器に偽の NMEA データ（AIS/GPS信号）を送りつけます。
